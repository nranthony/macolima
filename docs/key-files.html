What's hot vs. cold, concretely

The per-up seeding contract in profile.sh:ensure_state() reads only three input artifacts from the repo:

  1. config/claude-settings.json → copied to claude-home/settings.json if the per-profile copy doesn't exist (profile.sh:174).
  2. config/skills/<name>/ → each dir copied to claude-home/skills/<name> if absent (profile.sh:186).
  3. config/zshrc-snippet.sh is not re-read at up — only by bootstrap.sh.
Everything else (compose file, seccomp, squid config, allowlist) the scripts don't read directly — they hand the paths to docker compose, which mounts them live. Edits to those take effect on container recreate (no script change needed), while edits to config/claude-settings.json only affect new profiles — existing profiles keep their seeded copy. That asymmetry is the most important thing to internalize.

OUTPUT — what those scripts write to the data drive

Only the files ensure_state() and wipe/reset actually touch. Everything else under claude-home/ (sessions, projects, file-history, paste-cache, backups, telemetry…) is Claude Code writing its own state at runtime — the macolima scripts never read it.

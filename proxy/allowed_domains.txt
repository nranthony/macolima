# =============================================================================
# Outbound domain allowlist for Claude Code sandbox
# =============================================================================
# One domain per line. Leading dot (e.g. .github.com) matches all subdomains.
# Comments use `#` at the START of a line. INLINE comments after a domain
# WILL BREAK Squid — `dstdomain` treats the whole line (including `#` and
# any trailing text) as one hostname string. Keep notes on their own lines.
#
# Three lifecycle tiers, top to bottom:
#   - ALWAYS ON: every profile needs these. Never comment out.
#   - PLANNING-MODE: short-lived installs (pypi/npm/git/OS). Uncomment while
#     you (the human) install or clone, recomment before handing the session
#     back to an autonomous agent.
#   - PROJECT-PERSISTENT: long-lived per-project sets (paperbridge, wearables
#     vendors, OA publishers, archive). Uncomment for the duration of a
#     project; recomment when the project is dormant.
#
# After any edit, restart that profile's Squid:
#   PROFILE=<p> COMPOSE_PROJECT_NAME=macolima-<p> docker compose restart egress-proxy
#
# Blocks are tagged like `[name]` for grep — e.g. `grep "\[paperbridge\]" allowed_domains.txt`.
# To bulk-toggle a block: select its lines and `s/^# //` (uncomment) or
# `s/^/# /` (recomment). Blank lines inside blocks are intentional visual
# chunking — Squid ignores them.
# =============================================================================

# --- Paperbridge [paperbridge] ---
# Load-bearing for the paperbridge library. Without these every CrossRef /
# PubMed / OpenAlex / Unpaywall / EuropePMC call returns 403 from Squid and
# paperbridge silently produces empty results. Used by any project that does
# literature lookups.
#
# Covers: bibliographic metadata APIs (CrossRef, OpenAlex, Unpaywall, BASE),
# NCBI E-utilities (the JSON API — distinct from pubmed.ncbi.nlm.nih.gov which
# is HTML), EuropePMC, preprint servers (arxiv, biorxiv, medrxiv, chemrxiv),
# DOI resolver (302s to publishers — see [oa-publishers] block for destinations).
# Uncomment www.zotero.org only if using ZoteroClient for library sync.
# api.crossref.org
# api.openalex.org
# api.unpaywall.org
# api.base-search.net

# eutils.ncbi.nlm.nih.gov

# --- Open-access publisher destinations for doi.org redirects [oa-publishers] ---
# doi.org returns 302 to a publisher. These cover OA publishers most likely to
# host wearables/biosensor research. Gated publishers (Wiley, Springer, Nature,
# Sage, T&F, OUP) are intentionally omitted — rely on UnpaywallClient (via
# paperbridge) to find an OA copy on PMC / arXiv / a repository already in

# =============================================================================
# Outbound domain allowlist for Claude Code sandbox
# =============================================================================
# One domain per line. A leading dot (e.g. .github.com) matches all subdomains.
# Keep this list minimal — fewer domains = smaller attack surface.
#
# Two-phase usage pattern:
#   - AUTONOMOUS runs (agent driving): only this default list — Anthropic + VS
#     Code Marketplace. No git remotes, no package registries. Agent can't
#     clone, install, or push to anything. Web search still works because
#     WebSearch/WebFetch run server-side on Anthropic's infra.
#   - PLANNING runs (you driving): uncomment the "planning only" section
#     below, restart Squid (`docker compose restart egress-proxy` via
#     scripts/profile.sh), do your installs/clones/pushes, then re-comment
#     and restart before firing off any autonomous work.
# =============================================================================

# --- Claude / Anthropic (always on) ---
# Specific subdomains only — broad wildcards (`.anthropic.com`, `.claude.com`,
# `.claude.ai`) were dropped per security audit M3. A future subdomain that
# hosts user-controllable content under one of those parents would otherwise
# become an exfil channel. Add specific hosts here as Claude Code needs them;
# the error from the proxy (403) tells you which to add.
api.anthropic.com
console.anthropic.com
statsig.anthropic.com
api.claude.com
platform.claude.com 
claude.ai

# --- VS Code Marketplace (IDE integration) ---

# no known third-party-owned subdomain under that parent.
marketplace.visualstudio.com
.vscode-unpkg.net
update.code.visualstudio.com

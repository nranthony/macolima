# =============================================================================
# Wearables research crawl — auxiliary domain allowlist (REFERENCE FILE)
# =============================================================================
# This file is NOT loaded by Squid. squid.conf reads `allowed_domains.txt` only.
#
# Usage:
#   1. Before a wearables crawl run, copy the relevant blocks below into
#      `allowed_domains.txt` (uncommented), keeping the per-block headers so
#      future-you knows what to remove.
#   2. Restart Squid:
#        PROFILE=therapod COMPOSE_PROJECT_NAME=macolima-therapod \
#          docker compose restart egress-proxy
#   3. After the crawl, remove the inserted blocks and restart Squid again.
#
# Two distinct categories below — they have different lifecycles:
#   - PAPERBRIDGE APIs: needed any time paperbridge runs. Long-lived if the
#     therapod profile is doing literature work regularly.
#   - VENDOR DOCS / RESEARCH ARCHIVE: needed only during the active crawl run.
#     Recomment when the run completes.
# =============================================================================
